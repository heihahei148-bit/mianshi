Briefing // 规则设计优先

把合租争吵变成可执行的公共规则

针对卫生间早高峰、厨房使用、客厅占用和清洁标准四类高频冲突，先建立低成本、可执行、可复盘的共同边界。

选择方向：A 规则设计优先
A
金融实习生 早出晚归，早高峰卫生间和稳定休息是刚需。
B
程序员 昼夜颠倒，夜间活动需要被低打扰规则约束。
C
自由职业者 长期在家，客厅隔断带来公共空间边界争议。
01 // Conflict Map

冲突关系图

这不是单点矛盾，而是时间、空间、标准和责任四组约束互相叠加。方案先处理最容易反复爆发的公共资源边界。

A：早高峰刚需 早出晚归，卫生间使用窗口固定，迟到成本高。
卫生间排他 同一时间只能一人使用，需要时间表和超时成本。
规则 1：高峰排班 工作日 7:00-8:30 固定时段，前一晚可协商调换。
C：客厅长期占用 个人工作空间与公共空间属性发生冲突。
空间边界不清 谁能用、用多久、能否会客，没有共同答案。
规则 2：预约限时 客厅默认公共，C 可预约办公，但保留共享时段。

03 // Execution

执行闭环

这套方案不追求一次到位，而是通过试运行让规则从“写出来”变成“跑起来”。

第 1 天：确认边界 确认客厅隔断是否被合同认可；若未认可，按公共空间处理。
第 2 天：贴出规则 把一页规则纸发群并贴在公共区域，默认全员可见。
第 3-7 天：试运行 只记录超时、未复位、客厅占用三类事实，不在现场争论。
第 7 天：复盘一条 周日 15 分钟复盘，只调整最痛的一条规则，避免越改越复杂。
04 // Trade-off

取舍与风险

舍为什么不先做工具

数字化工具只能承载规则，不能替代共识。如果一开始做共享日历或 Notion，三人仍会争论谁有优先权、什么叫清理干净。

舍为什么不先做反馈闭环

当前冲突已经高频发生，先需要止血。没有基础规则，复盘容易变成情绪吐槽，无法产生可执行改动。

险C 对客厅归属有异议

先确认合同边界。如果合同认可客厅隔断，则只约束公共时段；如果不认可，则按公共空间重新划定使用权。

险有人不执行

不设置高惩罚，采用低成本违约后果：补做清洁、失去下次优先权、下次复盘优先讨论该条。
